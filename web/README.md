# Arizona Rule of Law Web App

This Next.js app is the canonical presentation experience for the Arizona Rule of Law Ambassador Program. It reads structured content generated by the Python pipeline from `web/content/generated/web-experience.json` and serves downloadable companion files from `web/public/downloads/`.

## Routes

- `/` landing page and entry point
- `/presentation` public presentation mode
- `/certification` self-guided certification and knowledge-check tab
- `/resources` Arizona partner resources and all downloadables
- `/presenter` scripts, talk tracks, guides, and presenter prep
- `/presentation/speaker` direct presenter-mode route for live delivery
- `/toolkit` legacy alias of the presenter workspace
- `/studio` static note page in the GitHub Pages build; deploy Studio separately if you need browser-based editing

## Development

```powershell
npm install
npm run dev
```

Validate:

```powershell
npm run test
npm run build
npm run test:e2e
```

## Static export for GitHub Pages

This app is configured to export as a static site so it can be shared without a running Node server.

Build locally:

```powershell
$env:NEXT_PUBLIC_BASE_PATH = "/your-repo-name"
npm install
npm run build
Remove-Item Env:NEXT_PUBLIC_BASE_PATH
```

The static site will be written to:

- `web/out/`

Notes:

- Set `NEXT_PUBLIC_BASE_PATH` to your GitHub repository name with a leading slash.
- The presentation, toolkit, resources, and downloads are included in the static export.
- Sanity Studio is not bundled into the shared GitHub Pages site.

## GitHub Pages deployment

A workflow is included at:

- `.github/workflows/deploy-pages.yml`

After pushing to GitHub:

1. Enable GitHub Pages for the repository.
2. Set the source to `GitHub Actions`.
3. Push to `main`.

The workflow builds `web/out/` and publishes it to GitHub Pages automatically.

## Content flow

1. Run the Python pipeline from the repo root.
2. The `web_experience` stage writes the canonical web payload to `outputs/web_export/`.
3. The same stage syncs the generated JSON into `web/content/generated/`.
4. Downloadable manual, handout, field guide, and LMS files are copied into `web/public/downloads/`.

## Sanity overrides

Sanity is used for presentation-safe overrides and event-specific copy. Configure:

- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET`
- `NEXT_PUBLIC_SANITY_API_VERSION`
- `SANITY_API_READ_TOKEN`

The pipeline remains the source of truth for sourced substance. Sanity is intended for curated delivery-layer overrides, not silent replacement of authoritative claims.
